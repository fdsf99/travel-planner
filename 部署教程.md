# 部署上线完整教程

本教程分三部分：**① Web 网页端部署（Vercel，公网可访问）**、**② 微信小程序发布上线**、**③ 管理后台使用与常见报错**。

> 全程免费方案，部署后任何设备通过浏览器/微信均可访问，支持多人同时使用。

---

## 一、整体架构

```
                    ┌─────────────────────────────┐
   微信小程序  ────▶│                             │
   (前端)           │   Vercel 后端 + Web 静态页   │
                    │   https://xxx.vercel.app    │
   Web 浏览器  ────▶│                             │
   (公网访问)       └────────────┬────────────────┘
                                 │
                          ┌──────▼──────┐
                          │  Supabase   │
                          │  (数据库)    │
                          └─────────────┘
```

- **后端**：Express，部署到 Vercel，自带 HTTPS + 公网域名
- **Web 前端**：纯静态 HTML/JS，与后端**同域名**托管（`/public` 目录），无跨域问题
- **小程序**：导入微信开发者工具，请求同一个 Vercel 域名
- **数据库**：Supabase（PostgreSQL 托管，免费额度）

## 三、Step 1 — Supabase 数据库配置

1. 注册 Supabase → 新建项目（记下区域和密码）
2. 项目首页 → **Settings → API**，复制：
   - `Project URL`（形如 `https://xxxx.supabase.co`）
   - `anon public` key
3. 左侧 **SQL Editor** → 新建查询 → 粘贴本项目 `backend/database_schema.sql` 全部内容 → **Run** 执行
4. （RLS 已在 `fix_rls_policies.sql` 处理，按需执行）

> 保存好 URL 和 Key，下一步要用。

## 四、Step 2 — 后端 + Web 部署到 Vercel

### 方式 A：用 Vercel CLI（推荐，最简单）

1. **安装 CLI**
   ```bash
   npm i -g vercel
   ```

2. **进入后端目录**
   ```bash
   cd D:\旅游规划系统\backend
   ```

3. **登录**
   ```bash
   vercel login
   ```
   （用邮箱或 GitHub 登录）

4. **部署**
   ```bash
   vercel --prod
   ```
   - Project name：回车用默认（如 `travel-planner-backend`）
   - 其他全部回车用默认

   部署完成会输出一个地址，如：
   ```
   https://travel-planner-backend.vercel.app
   ```

5. **配置环境变量**（关键！）
   ```bash
   vercel env add SUPABASE_URL production
   # 粘贴: https://xxxx.supabase.co

   vercel env add SUPABASE_KEY production
   # 粘贴: 你的 anon key

   vercel env add AMAP_KEY production
   # 粘贴: 高德 key(没有可跳过,会用数据库已有景点)

   vercel env add DASHSCOPE_API_KEY production
   # 粘贴: 通义千问 key(没有可跳过,AI 功能会降级)

   vercel env add ADMIN_USERNAME production
   # 输入: admin

   vercel env add ADMIN_PASSWORD production
   # 输入: 你的强密码(务必修改默认值!)

   vercel env add JWT_SECRET production
   # 输入: 任意长随机字符串
   ```
   每条命令会提示输入值，粘贴即可。

   > 也可以在网页 https://vercel.com → 你的项目 → **Settings → Environment Variables** 图形化添加。

6. **重新部署让环境变量生效**
   ```bash
   vercel --prod
   ```

7. **验证**
   - 浏览器打开 `https://你的域名.vercel.app/health` → 返回 `{"status":"OK",...}`
   - 打开 `https://你的域名.vercel.app/` → 看到 Web 首页 ✅
   - 打开 `https://你的域名.vercel.app/login.html` → 管理后台登录页 ✅

### 方式 B：GitHub 关联自动部署

1. 把整个项目 push 到 GitHub 仓库
2. Vercel 网页 → **New Project** → Import 你的仓库
3. **Root Directory** 设为 `backend`
4. **Environment Variables** 填入上面 7 个变量
5. **Deploy** → 之后每次 push 代码自动重新部署

## 六、Step 4 — 微信小程序发布上线

### 6.1 修改 API 地址

编辑 `miniprogram/app.js`，把 `apiBaseUrl` 改为你的 Vercel 域名：

```javascript
globalData: {
  apiBaseUrl: 'https://你的域名.vercel.app/api',  // 改成这一行
  ...
}
```

### 6.2 配置合法域名（必须，否则线上无法请求）

1. 登录 [微信公众平台](https://mp.weixin.qq.com)（用你的小程序账号）
2. 左侧菜单 → **开发管理 → 开发设置 → 服务器域名**
3. 在 **request 合法域名** 中添加：
   ```
   https://你的域名.vercel.app
   ```
   （注意：必须 https，不能带路径，每月可改 50 次）
4. 保存

### 6.3 上传代码

1. 打开**微信开发者工具**，导入 `miniprogram` 目录
2. 确认 AppID 是你的正式 AppID（非测试号）：右上角"详情" → "项目详情"
3. 点击工具栏 **上传** 按钮
4. 填写**版本号**（如 `1.0.0`）和**项目备注** → 确定

### 6.4 提交审核 + 发布

1. 回到微信公众平台网页 → **管理 → 版本管理**
2. 在**开发版本**里找到刚上传的版本 → 点 **提交审核**
3. 填写审核信息（功能页面截图、测试账号等）
4. 提交后通常 **1~7 天**审核完成（结果通过站内信/邮件通知）
5. 审核通过后 → 在**审核版本**点 **发布**
6. 发布后用户即可在微信内**搜索小程序名称**使用

### 6.5 体验版（审核前给团队测试）

提交审核前，可先在 **管理 → 成员管理** 添加"体验成员"，开发版本点**选为体验版**，成员扫码即可体验。

## 八、常见报错与解决

### Q1: `Supabase connection error` / `fetch failed`
- **原因**：`SUPABASE_URL` 或 `SUPABASE_KEY` 没配 / 配错 / Supabase 项目暂停
- **解决**：检查 Vercel 环境变量；Supabase 免费项目 7 天不用会暂停，登录后台点 **Restore** 恢复
- **本系统已做降级处理**：连不上数据库仍可启动，但行程 CRUD 不可用

### Q2: 小程序提示"不在以下 request 合法域名列表中"
- **原因**：未在公众平台配置域名，或开发工具没勾"不校验合法域名"
- **解决**：
  - 线上：去公众平台加 `https://你的域名.vercel.app` 到 request 合法域名
  - 调试：开发者工具 → 详情 → 本地设置 → 勾"不校验合法域名、TLS 版本及 HTTPS 证书"

### Q3: AI 生成行程失败 / 降级为规则推荐
- **原因**：`DASHSCOPE_API_KEY` 或 `ZHIPU_API_KEY` 未配 / 失效 / 余额不足
- **解决**：检查 Vercel 环境变量；通义千问新用户有免费额度，确认 key 有效

### Q4: 管理后台登录提示"用户名或密码错误"
- **原因**：环境变量 `ADMIN_USERNAME` / `ADMIN_PASSWORD` 未设或值不对
- **解决**：Vercel → Settings → Environment Variables 添加并重新部署
- 默认 `admin / admin123`，但仅在你**没设置环境变量**时才生效

### Q5: Vercel 部署后 404 / 页面空白
- **原因**：Root Directory 没选对，或 `public` 目录没上传
- **解决**：确认入口是 `backend/server.js`，`backend/public/` 完整上传；用 CLI 在 `backend` 目录下执行 `vercel --prod`

### Q6: 小程序地图不显示 / 景点没坐标
- **原因**：数据库景点缺 `longitude/latitude`，或高德 key 未配导致没拉到 POI
- **解决**：配置 `AMAP_KEY`；或在 Supabase 手动给 attractions 表补坐标

### Q7: `vercel` 命令找不到
- **原因**：未全局安装或 PATH 问题
- **解决**：`npm i -g vercel`；Windows 用户重启终端

### Q8: 环境变量改了不生效
- **原因**：Vercel 环境变量需重新部署才生效
- **解决**：网页端点 **Redeploy**，或 CLI 执行 `vercel --prod`

## 九、域名自定义（可选，让网址更专业）

Vercel 免费子域名 `xxx.vercel.app` 已够用。如需自有域名：

1. Vercel → 项目 → **Settings → Domains** → 添加你的域名
2. 按提示到域名服务商添加 CNAME 记录指向 `cname.vercel-dns.com`
3. Vercel 自动签发 HTTPS 证书
4. 同步更新小程序 `app.js` 和公众平台合法域名
